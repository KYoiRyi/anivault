# AniVault 级联设计美学与视觉准则 (Design Guidelines)

AniVault 致力于达成克制、极致冷静但又不失物理响应感的高级审美理念。本准则明确了 UI/UX 领域内的最高优先级信条，主要融合了“Apple 无彩极简规范”以及“FlClash 质感灰阶体系”。

## 1. 无色系极简基础 (Monochrome Palette)

- **绝对的黑白分级**：摒弃一切具有明显偏向的非功能性彩色（包括且不限于荧光色、高饱和度紫色、赛博渐变光效）。全局采用最为理性的黑、白、灰色域（基于 `#000000` 到纯白之间的色阶过渡）。
- **Apple 式结构设计**：页面的基本构型需要遵循 Apple 原生应用的重逻辑。诸如：`32px` 大字重左对齐头部 `SliverAppBar`；大量引入留白结构而不是强行用复杂的背景或者色块进行区域填充。

## 2. 几何形变与圆弧准则 (Squircles & Shapes)

- **避免尖锐与绝对纯圆**：在 AniVault 中，坚决排斥尖锐刺眼的直角卡片（`0px`），同时也要避免过时且随意的纯圆形（除非是特殊悬浮指示符）。
- **FlClash 高级弧线**：一切卡牌基底、图标按钮，均需严格采用类似 `RoundedRectangleBorder(borderRadius: 14)` 或是 `16px`、`24px` 的超椭圆/大圆角矩形式样，从而传达温润且工业的标准美学。

## 3. Tonal 材质与环境光阶响应 (Material 3 Tonal Surface)

这是体现“灵动感”的最核心要素。极简绝不等于死板。
- **背景面灰阶流转**：禁止使用极度夸张的硬阴影投射和高度抬升。当构建基础交互模块时（列表按钮、视频卡），利用 `colorScheme.surfaceContainerLow` 承载，只在其受到交互（Hover / 选中）时微调至 `surfaceContainerHighest` 或搭配内敛但明确的底层边框色变（`surfaceContainerHighest`）。
- **流动的点击反馈 (InkWell Fluidity)**：绝不采用生硬死切的变色点击。完全依赖官方底层的 Material 水波纹组件（InkWell）。每一次接触屏幕或者鼠标按压，都必须产生极易察觉但不刺眼的低饱和灰色涟漪外扩荡漾。
- **按钮范式**：无论是增加媒体还是播放暂停控制中心，全线统一切换到 `FilledButton.tonal` / `IconButton.filledTonal`。利用这种 Tonal （原色调低对比）风格带来的系统原生水声物理质感，统御界面的所有焦点控件。

## 4. 突破边界的环境光效 (Cinematic Ambient Progress)

界面的框架是极简且受限的，但内容的呈现是无界的。
- **边缘光韵环境**：在播放进度条上，我们取消了独立而僵硬的实体容器带。采用 0 边距紧贴视界极底的 “无边缝光轨”。
- **立体向上映射**：在极暗视频画面的衬托底色下，该底层光轨须通过巨大的负向虚阴影半径（例如 `Offset(0, -20)`, `blurRadius: 40`），将自身的播放流光向原片画幅层层递进地“向上发散混合”，营造出一层浑然天成的物理环境光（Ambilight）特效感。
